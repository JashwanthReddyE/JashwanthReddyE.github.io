§ 04 / Featured Projects

Two live pipelines I shipped solo, end-to-end.

LIVE · Azure Serverless · Open Source

Crypto Sentinel — AI Sentiment Pipeline

CoinGecko + CryptoPanic → 6-signal weighted scoring → daily BUY / WATCH / AVOID picks

Live Dashboard View on GitHub

An Azure Functions pipeline that scores 24 crypto assets on a daily cadence and ranks them BUY / WATCH / AVOID. A timer-triggered function pulls live prices, market caps, and trending lists from CoinGecko, plus headline sentiment from CryptoPanic's public feed. The scoring engine blends six signals — news sentiment, three momentum windows, volume ratio, trending bonus — into a single 0–100 score, exposed through two HTTP endpoints and a self-contained Vercel dashboard. The whole thing runs at ~$0.03/month on Consumption + LRS storage.

Azure Functions v4 Python 3.11 CoinGecko CryptoPanic Timer Trigger HTTP Trigger LRS Storage Vercel pytest · 144 tests
Architecture
Scoring engine · 6 signals → 0–100 → BUY / WATCH / AVOID
Decisions under constraint
SIGNAL WEIGHT

News sentiment capped at 30%.

Letting NLP dominate the score would over-react to a single bullish headline. Combining sentiment with three momentum windows (24h / 7d / 30d) plus volume ratio and a trending bonus smooths the picks against single-source manipulation.

COST CEILING

Whole pipeline at ~$0.03/month.

Picked Consumption plan + LRS storage instead of always-on App Service. A daily timer plus on-demand HTTP fits comfortably inside the Functions free grant; the only billable resource is a few KB of state in storage.

OFFLINE MODE

Every endpoint takes test_data=true.

A flag swaps live API calls for canned JSON in tests/. CI runs all 144 pytest cases hermetically without burning the CoinGecko demo key, and reviewers can demo the dashboard offline.

LIVE · AWS Free Tier · Open Source

Stock Sentiment & Forecast Pipeline

News ingest → LLM sentiment scoring → 5-day price forecast → Streamlit dashboard

View on GitHub

A complete AWS data pipeline scoped to the Free Tier. EventBridge fires a Lambda every 30 minutes that pulls financial news from Finnhub. An S3:ObjectCreated event triggers a second Lambda that scores every headline through Bedrock Claude Haiku (−1 bearish → +1 bullish) and writes Parquet to silver. A daily forecast Lambda pulls 60 days of prices from Alpha Vantage, joins the LLM sentiment via Athena, projects a 5-day trend, and asks Claude to write a plain-English market brief. A Streamlit dashboard reads it all back. Everything provisioned with Terraform.

AWS Lambda S3 Medallion Bedrock Claude Haiku Glue + Athena EventBridge Streamlit Plotly Terraform Python 3.12 pytest + moto GitHub Actions
Architecture
Decisions under constraint
API SELECTION

Yahoo Finance silently blocks AWS IPs.

Every yfinance tutorial assumes Yahoo will answer. From a Lambda it doesn't — calls just time out. Swapped to Finnhub for news and Alpha Vantage for prices, splitting two free tiers to fit both quotas.

IAM GOTCHA

Athena needs GetBucketLocation, separately.

PutObject grants aren't enough — Athena does a region pre-flight that requires its own permission, returning a misleading "bucket does not exist" error if it's missing. Documented in the README so the next person saves the hour.

RESILIENCE

Graceful degradation beats happy path.

Alpha Vantage rate-limits at 1 req/sec — burst the loop and 9 of 10 tickers silently lose data. Sleep between calls, log the API's verbatim message on rate-limit, default sentiment to 0 if Athena is empty. Worst-case run still ships output.
